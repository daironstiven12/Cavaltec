Resumen Ejecutivo

Que es CAVALTEC

CAVALTEC es una plataforma web disenada para ayudar a las empresas colombianas a evaluar, monitorear y mejorar su cumplimiento normativo en materia de proteccion de datos personales, cumpliendo con la Ley 1581 de 2012 y el Decreto 1377 de 2017.

Para que sirve

Evaluacion

Cuestionarios estructurados con 20 preguntas en 4 categorias para medir el nivel de cumplimiento de una empresa.

Monitoreo

Dashboards con graficos de tendencia, resultados por categoria y seguimiento historico de evaluaciones.

Asistente IA

Chatbot inteligente que responde exclusivamente sobre temas de cumplimiento normativo usando OpenRouter + Gemma 4.

Reportes

Generacion y descarga de reportes detallados con recomendaciones de mejora priorizadas por impacto.

Problema que resuelve

Las empresas colombianas enfrentan complejidades para interpretar y aplicar la Ley 1581 de 2012. CAVALTEC automatiza el proceso de evaluacion de cumplimiento, ofrece recomendaciones personalizadas impulsadas por IA y centraliza la gestion de datos personales en una plataforma intuitiva, reduciendo el riesgo de sanciones y mejorando la confianza de los titulares de datos.

Enfoque: La plataforma esta disenada con un enfoque practico: cuestionarios validados por expertos, IA especializada en cumplimiento, y una interfaz accesible para usuarios no tecnicos.

Arquitectura del Sistema

Arquitectura cliente-servidor con separacion clara de capas y comunicacion asincrona.

Flujo de Datos

  1. El Frontend (React) envia requests HTTP/JSON al Backend a traves de Axios con interceptores de autenticacion.
  2. El Backend (FastAPI) valida el JWT, ejecuta la logica de negocio y consulta MySQL a traves de SQLAlchemy async.
  3. Para consultas de IA, el Backend reenvia el mensaje a OpenRouter API usando httpx async client.
  4. OpenRouter procesa con Gemma 4 26B y retorna la respuesta que el Backend formatea y devuelve al Frontend.

Por que Docker?

Docker permite replicar el entorno exacto en cualquier maquina, eliminando el problema "funciona en mi PC". Con docker-compose, un solo comando levanta los 3 servicios (backend, MySQL, phpMyAdmin) con redes aisladas, volumenes persistentes y healthchecks configurados.

Autenticacion y Roles

Flujo JWT

  1. El usuario envia email + password al endpoint POST /auth/login.
  2. El backend verifica las credenciales con Argon2 y retorna access_token + refresh_token.
  3. El Frontend almacena el token en localStorage y lo envia en cada request via header Authorization: Bearer.
  4. El backend valida el JWT en cada endpoint protegido usando dependency injection.

Asistente de IA

Como funciona

El asistente utiliza OpenRouter API para conectar con Google Gemma 4 26B, un modelo de lenguaje gratuito. El Backend envia el mensaje del usuario junto con un system prompt que restringe las respuestas exclusivamente a temas de cumplimiento normativo.

Despliegue en Vercel

El frontend React puede desplegarse en Vercel como aplicacion estatica.

Configuracion vercel.json

json{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite",
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}

Pasos

  1. Conectar el repositorio GitHub a Vercel.
  2. Configurar el root directory como frontend.
  3. Agregar variable de entorno VITE_API_URL con la URL del backend.
  4. Vercel detecta automaticamente Vite y ejecuta npm run build.
  5. Los rewrites en vercel.json redirigen todas las rutas al SPA.

Nota: El backend debe estar desplegado por separado (Railway, Render, Fly.io, o un VPS con Docker). El frontend solo necesita la URL del backend en la variable VITE_API_URL.

Decisiones Tecnicas

FastAPI sobre Flask/Django

Performance async nativa, documentacion Swagger automatica, validacion con Pydantic, dependency injection.

MySQL sobre PostgreSQL

Mayor compatibilidad con herramientas como phpMyAdmin, amplio soporte en hosting, suficiente para el volumen de datos.

Docker Compose

Un solo comando levanta todo el stack. Replicabilidad garantizada, sin configurar MySQL manualmente.

Argon2 sobre bcrypt

Ganador del Password Hashing Competition. Memory-hard, resistente a ataques GPU, configurable en memoria y tiempo.

OpenRouter sobre OpenAI

Acceso gratuito a multiples modelos. Sin tarjeta de credito. Cambio dinamico de modelos sin modificar codigo.

Recharts sobre Chart.js

API declarativa React-native, basado en D3.js, mejor integracion con React, tipado completo.
